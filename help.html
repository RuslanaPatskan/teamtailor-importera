❓ Часті запитання
Значки не з'являються на картках

1. Перевірте що введено API ключ Teamtailor: іконка 🎯 → ⚙️ Налаштування → «Перевірити ключ» → статус зелений.

2. Перезавантажте сторінку (F5).

3. Перевірте URL — розширення працює тільки на сторінках зі списком кандидатів (не на головній).

4. Якщо щойно встановили/оновили — перезавантажте розширення: chrome://extensions → toggle on/off.

Значок залишається зеленим хоча кандидат є в TT

Перевірка відбувається у фоні. Якщо значок залишається зеленим довше 5–10 секунд:

  • TT API може бути повільним або тимчасово недоступним.
  • Ім'я кандидата в TT може відрізнятися від імені на robota.ua/work.ua (наприклад, різний порядок слів або транслітерація).
  • Спробуйте натиснути «+TT» — при кліку відбувається повторна перевірка з реальними контактами (якщо вони є).
Ім'я та прізвище порожні в модалці

Можливі причини:

  • Кандидат приховав ім'я (приватний режим) — заповніть вручну.
  • API robota.ua/work.ua повільно відповів — спробуйте закрити модалку і натиснути «+TT» ще раз.
  • На hh.kz — ім'я може бути приховане до розгляду відгуку.

Навіть без імені імпорт можливий — заповніть поля вручну в модалці.

Помилка «Не залогінені на robota.ua» при завантаженні резюме

1. В налаштуваннях розширення → «🤖 robota.ua» → введіть правильний email і пароль.

2. Натисніть «Перевірити» — має бути зелений статус.

3. Переконайтесь що ви залогінені на robota.ua в браузері (деякі API потребують активної сесії).

Резюме (файл) не завантажується — work.ua

1. Переконайтесь що ви залогінені на work.ua в тому ж браузері.

2. Кнопка «⬇️ Завантажити» з'являється тільки у кандидатів з файлом резюме (не всі кандидати мають файл).

3. Перезавантажте сторінку work.ua та спробуйте знову.

Помилка 422 при імпорті

422 Unprocessable Entity — Teamtailor відхилив дані через невалідне поле.

  • Перевірте формат телефону (має починатись з +).
  • Перевірте email — має бути валідним або порожнім.
  • Якщо помилка повторюється — очистіть телефон або email і спробуйте знову.
Помилка 429 (rate limit) при масовому імпорті

429 Too Many Requests — Teamtailor тимчасово обмежив кількість запитів.

Розширення автоматично чекає та повторює запит. Під час масового імпорту це нормально — просто зачекайте, не закривайте вкладку.

Якщо помилка виникає постійно — зробіть перерву 2–3 хвилини між імпортами.

Кастомні поля (posil. на profil) не заповнюються

1. Переконайтесь що поля створені в TT: Settings → Custom Fields → Candidates.

2. Тип поля — Text (не URL, не Multi-select).

3. API-назва поля в налаштуваннях розширення має точно збігатись з API Name у TT. Регістр важливий: robota_ua ≠ Robota_UA.

Розширення не активується на сторінці

Перевірте URL сторінки:

  • robota.ua: robota.ua/my/vacancies/{ID}/candidates
  • work.ua: work.ua/employer/my/applicants/
  • hh.kz: hh.kz/employer/vacancy/{ID}/responses

Розширення не працює на головній сторінці, сторінці пошуку вакансій або інших розділах. Перейдіть саме на сторінку відгуків/кандидатів.

Як імпортувати кандидатів з кількох вакансій?

Відкрийте кожну вакансію в окремій вкладці та робіть імпорт по черзі. Розширення підтримує кілька вкладок одночасно.

В модалці можна вручну змінити вакансію для кожного кандидата (не обов'язково відповідатиме поточній сторінці).

Яка різниця між «Оновити профіль» і «Все одно додати»?

Оновити профіль — знаходить існуючого кандидата в TT та додає відсутні дані (телефон, email, теги, посилання). Наявні поля не змінюються.

Все одно додати — створює нового кандидата навіть якщо схожий вже є. Обережно — це створює дублікат у TT.
